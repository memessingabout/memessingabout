History & Versions of HTML

HTML (HyperText Markup Language) is the standard language for creating webpages. Developed by Tim Berners-Lee in 1990, it has evolved through multiple versions:

  • HTML 1.0 (1990): Basic structure with tags like <h1>, <p>, and <a>.
  • HTML 2.0 (1995): Standardized by IETF, added forms (<form>, <input>).
  • HTML 3.2 (1997): Introduced tables, applets, and text formatting.
  • HTML 4.01 (1999): Added stylesheets, frames, and better accessibility.
  • XHTML 1.0 (2000): XML-based HTML, stricter syntax.
  • HTML5 (2014): Current standard, added semantic elements (<header>, <footer>), multimedia (<audio>, <video>), and APIs (e.g., Canvas, Web Storage).
  • HTML5.1, 5.2, Living Standard: Ongoing updates by WHATWG as a "living standard" (no fixed versions).

Key Notes: Use HTML5 for modern web development. Deprecated tags like <font> or <center> should be replaced with CSS.

Structural Elements

These elements define the structure and hierarchy of a webpage.

Element Description Syntax/Example
<html> Root element of the page. 
<html lang="en">...</html>
<head> Contains metadata, styles, and scripts. 
<head>
    <title>Page Title</title>
</head>
<body> Contains visible content. 
<body>
    <p>Content</p>
</body>
<header> Introductory content or navigation. 
<header>
    <h1>Site Title</h1>
</header>
<footer> Concluding content (e.g., copyright). 
<footer>
    <p>© 2025</p>
</footer>
<main> Main content of the page. 
<main>
    <p>Main content</p>
</main>
<nav> Navigation links. 
<nav>
    <ul>
        <li><a href="#">Home</a></li>
    </ul>
</nav>
<section> Groups related content. 
<section>
    <h2>Section</h2>
    <p>Content</p>
</section>
<article> Standalone content (e.g., blog post). 
<article>
    <h2>Post</h2>
    <p>Content</p>
</article>
<aside> Tangential content (e.g., sidebar). 
<aside>
    <p>Sidebar</p>
</aside>
<div> Generic block container. 
<div class="container">Content</div>
<h1> to <h6> Headings for hierarchy. 
<h1>Main Title</h1>
<h2>Subtitle</h2>

Notes: Use semantic elements for accessibility and SEO. Avoid <div> for semantic roles like navigation or headers.

Content Elements

These elements display or format text and inline content.

Element Description Syntax/Example
<p> Paragraph text. 
<p>This is a paragraph.</p>
<a> Hyperlink to pages, files, or anchors. 
<a href="https://example.com">Link</a>
<span> Generic inline container. 
<span class="highlight">Text</span>
<strong> Semantic bold (importance). 
<strong>Important</strong>
<em> Semantic italic (emphasis). 
<em>Emphasized</em>
<b> Visual bold (non-semantic). 
<b>Bold</b>
<i> Visual italic (non-semantic). 
<i>Italic</i>
<u> Underlined text. 
<u>Underline</u>
<s> Strikethrough text. 
<s>Old price</s>
<del> Deleted text (semantic). 
<del>Removed</del>
<mark> Highlighted text. 
<mark>Highlight</mark>
<sup> Superscript text. 
E = MC<sup>2</sup>
<sub> Subscript text. 
H<sub>2</sub>O
<code> Code snippet. 
<code>let x = 10;</code>
<q> Inline quotation. 
<q>Quote</q>
<blockquote> Block quotation. 
<blockquote>Long quote</blockquote>
<abbr> Abbreviation with tooltip. 
<abbr title="HyperText">HTML</abbr>
<cite> Reference to a work. 
<cite>The Great Gatsby</cite>
<time> Date/time. 
<time datetime="2025-10-21">Oct 21, 2025</time>
<kbd> Keyboard input. 
<kbd>Ctrl + S</kbd>
<var> Variable (e.g., in math/code). 
<var>x</var> = 10
<pre> Preformatted text (preserves whitespace). 
<pre>
function code() {
    console.log("Hello");
}
</pre>
<hr> Thematic break (horizontal line). 
<hr>
<br> Line break. 
Line 1<br>Line 2

Notes: Use semantic tags (<strong>, <em>) over presentational ones (<b>, <i>) for accessibility. Avoid <br> for spacing; use CSS margins.

Multimedia Elements

These elements embed images, audio, video, and other media.

Element Description Syntax/Example
<img> Embeds an image. 
<img src="image.jpg" alt="Description" width="100">
<audio> Embeds audio with controls. 
<audio src="audio.mp3" controls>
    <source src="audio.mp3" type="audio/mpeg">
    Fallback text
</audio>
<video> Embeds video with controls. 
<video src="video.mp4" controls width="300">
    <source src="video.mp4" type="video/mp4">
    Fallback text
</video>
<figure>, <figcaption> Image/diagram with caption. 
<figure>
    <img src="image.jpg" alt="Description">
    <figcaption>Caption</figcaption>
</figure>
<iframe> Embeds another webpage. 
<iframe src="https://example.com" width="560" height="315"></iframe>
<embed> Embeds external content (e.g., PDF). 
<embed src="file.pdf" type="application/pdf">
<object> Embeds objects (e.g., multimedia). 
<object data="file.pdf" type="application/pdf"></object>
<canvas> Drawable area for graphics (JavaScript). 
<canvas id="myCanvas" width="200" height="100"></canvas>
<svg> Scalable vector graphics. 
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="red"/>
</svg>

Notes: Always include alt for <img>, use multiple <source> for <audio>/<video>, and optimize media for performance.

Forms & Input Elements

These elements collect user input for forms and interactivity.

Element Description Syntax/Example
<form> Wraps input elements for submission. 
<form action="/submit" method="POST">
    <input type="text" name="name">
    <button type="submit">Submit</button>
</form>
<input> Various input types (text, email, etc.). 
<input type="text" name="name" required>
<input type="radio" name="option" value="yes">
<input type="file" name="upload">
<label> Associates text with inputs. 
<label for="name">Name:</label>
<input id="name" type="text">
<select>, <option> Dropdown menu. 
<select name="choice">
    <option value="opt1">Option 1</option>
</select>
<textarea> Multi-line text input. 
<textarea name="message" rows="4"></textarea>
<button> Triggers actions (submit, reset, etc.). 
<button type="submit">Submit</button>
<fieldset>, <legend> Groups form controls. 
<fieldset>
    <legend>Info</legend>
    <input type="text">
</fieldset>
<datalist> Autocomplete suggestions. 
<input list="options">
<datalist id="options">
    <option value="Opt1">
</datalist>
<output> Displays calculation results. 
<output name="result">0</output>
<progress> Shows task progress. 
<progress value="50" max="100"></progress>
<meter> Scalar value in a range. 
<meter value="75" min="0" max="100">75%</meter>

Notes: Use <label> for accessibility, validate inputs with required or pattern, and use POST for sensitive data.

Interactive Elements

These elements enable interactivity beyond forms.

Element Description Syntax/Example
<details>, <summary> Collapsible content. 
<details>
    <summary>Click to expand</summary>
    <p>Content</p>
</details>
<dialog> Modal dialog/popup. 
<dialog id="myDialog">
    <p>Modal content</p>
</dialog>
<table> Tabular data (rows/columns). 
<table>
    <tr>
        <th>Header</th>
        <td>Data</td>
    </tr>
</table>
<ul>, <ol>, <li> Unordered/ordered lists. 
<ul>
    <li>Item 1</li>
</ul>
<ol>
    <li>Step 1</li>
</ol>
<dl>, <dt>, <dd> Description list. 
<dl>
    <dt>Term</dt>
    <dd>Description</dd>
</dl>

Notes: Use scope or headers for accessible tables. Ensure lists are semantic (e.g., <ul> for non-sequential items).

Global Attributes

These attributes can be applied to any HTML element.

Attribute Description Example
class Assigns CSS classes. 
<div class="container">
id Unique identifier. 
<div id="main">
style Inline CSS (avoid overuse). 
<div style="color: red;">
data-* Custom data attributes. 
<div data-user-id="123">
lang Specifies language. 
<html lang="en">
tabindex Controls keyboard focus order. 
<div tabindex="0">
contenteditable Allows editing content. 
<div contenteditable="true">
aria-* Accessibility attributes (e.g., aria-label). 
<button aria-label="Close">X</button>
onclick, onchange, etc. Event handlers for JavaScript. 
<button onclick="myFunction()">Click</button>

Notes: Use ARIA attributes for accessibility. Avoid inline style and event handlers; prefer external CSS/JavaScript.
